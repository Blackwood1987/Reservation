# 수동 QA 체크리스트

## 1. 목적
- 배포 전 주요 사용자 흐름과 화면 노출 상태를 빠르게 점검한다.
- 자동 테스트로 잡기 어려운 드래그, 리사이즈, 모바일 레이아웃, 권한 노출 문제를 확인한다.

## 2. 기본 확인
- 최신 배포본을 강력 새로고침(`Ctrl+F5`) 후 확인한다.
- 콘솔 오류가 없는지 확인한다.
- 로그인/게스트/관리자 권한별 화면이 의도대로 보이는지 확인한다.

## 3. 로그인 및 권한
- 작업자 로그인: 대시보드, 예약 관리, 월간 일정, 사용자 메뉴얼 탭만 노출되는지 확인한다.
- 관리자 로그인: 관리 탭과 관리자 전용 편집 기능이 노출되는지 확인한다.
- 게스트: 읽기 전용 흐름만 가능하고 관리자 기능이 숨겨지는지 확인한다.

## 4. 대시보드
- Site/Room 트리와 상세 패널이 정상 동기화되는지 확인한다.
- 실시간 타임라인 정렬과 현재 시각선이 정상인지 확인한다.
- CRF 장비가 타임라인 상단 규칙대로 표시되는지 확인한다.
- 라이브 ON/OFF, 날짜 표시, 툴팁이 정상 동작하는지 확인한다.

## 5. 예약 관리
- 예약 생성, 수정, 삭제 기본 흐름이 동작하는지 확인한다.
- 작업자 본인 예약만 이동/수정 가능하고, 관리자 권한은 더 넓게 동작하는지 확인한다.
- 리사이즈 프리뷰, 의도잠금, 중복 차단이 정상인지 확인한다.
- 장소 그룹 스크롤과 장비명 표시가 잘리지 않는지 확인한다.

## 6. 모바일
- 모바일 대시보드와 예약 관리가 단순화된 UX로 표시되는지 확인한다.
- 예약 관리의 카테고리 필터와 슬롯 상세 카드가 정상 동작하는지 확인한다.
- 모바일 예약 버튼이 선택 슬롯 시간을 이어받는지 확인한다.
- 메뉴 드로어/레일 전환과 반응형 레이아웃 깨짐이 없는지 확인한다.

## 7. 관리자 기능
- Site/Room 생성/수정/배치 편집이 정상 동작하는지 확인한다.
- 장소 유지보수 예약 일괄 등록/수정/삭제가 관리자에게만 노출되는지 확인한다.
- 사용자 메뉴얼 편집 내용이 작업자용 탭에 정상 반영되는지 확인한다.
- 운영 이력/백업/보고서 화면이 정상 노출되는지 확인한다.

## 8. 배포 전 최소 체크
- `node --check app.js`
- `node tests/run-tests.mjs`
- `node tests/check-text-integrity.mjs`
- `git diff --check`
