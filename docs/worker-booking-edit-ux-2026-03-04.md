# 작업자 예약 수정 UX 개선 (2026-03-04)

## 변경 전/후 대비
| 구분 | 변경 전 | 변경 후 | 영향 파일 |
|---|---|---|---|
| 작업자 예약 수정 | 작업자는 예약 블록 드래그 이동만 가능(상세 수정 진입 어려움) | 예약 블록 클릭으로 수정 모달 진입 가능(본인 예약 한정) | `app.js`, `app.html` |
| 예약 모달 모드 | 등록 전용 | 등록/수정 모드 분리(제목/버튼 문구/반복예약 표시 제어) | `app.js`, `app.html` |
| 드래그/리사이즈 UX | 리사이즈 핸들 히트 영역이 작아 커서 깜빡임 발생 가능 | 핸들 영역 확대 + 그립 표시 + 클릭/드래그 충돌 억제 로직 추가 | `app.js`, `styles.css` |
| 작업자 리사이즈 권한 | 관리자 중심 | 본인 예약이면 작업자도 리사이즈 가능 | `app.js`, `styles.css` |

## 안정성 처리
- 드래그 시작/리사이즈 시작 직후 클릭 이벤트를 잠시 억제하여 오작동(의도치 않은 모달 오픈) 방지
- 수정 저장 시 중복 예약/운영시간/목적 제한 재검증
- 자동 소독 연동 예약은 수정 후 위치가 바뀌면 기존 건 정리 후 신규 위치에 재생성

## 검증
- `node --check app.js`
