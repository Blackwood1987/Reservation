# 운영/작업 이력 백업 및 이관 가이드

## 1) 목적
- 데모 운영 기간에는 운영 이력과 관리자 작업 이력을 관리자가 수동 백업한다.
- 추후 IT 부서 이관 시 영구 저장으로 전환할 수 있도록 기준 절차를 남긴다.

## 2) 현재 데모 운영 정책
- 백업 주체: 관리자 계정
- 백업 방식: `관리 > 운영 이력 및 보고` 화면에서 수동 다운로드
- 권장 주기:
  - 일일 1회 이상 (업무 종료 전)
  - 예약 변경이 많은 날은 중간 1회 추가
- 보관 기준:
  - 팀 공유 저장소에 날짜 폴더 단위로 저장
  - 최소 30일 보관 (내부 정책에 맞춰 조정)

## 3) 수동 백업 절차
1. `관리 > 운영 이력 및 보고`로 이동
2. `리포트 날짜` 선택
3. `이력 새로고침` 실행
4. 아래 버튼 순서대로 실행
   - `운영 이력 CSV 백업`
   - `작업 이력 JSON 백업`
5. 파일명/날짜 확인 후 팀 저장소 업로드

참고: 운영 이력 CSV는 화면 필터와 무관하게 선택 날짜의 전체 이력을 내보낸다.

## 4) 백업 파일 의미
- 운영 이력 CSV
  - 상태(확정/삭제/반려), 시간, 사유 포함
  - 감사/일일 리포트 근거 데이터
- 작업 이력 JSON
  - 관리자 화면 주요 작업 로그
  - 변경 추적 및 장애 복기 용도

## 5) IT 이관 시 영구 저장 권장안
- 대상 시점: 백엔드 전환 및 IT 운영 이관 단계
- 권장 저장 구조:
  - `auditLogs`: 예약 변경 이벤트(생성/수정/삭제/반려) append-only 저장
  - `adminActions`: 관리자 작업 로그 append-only 저장
  - TTL 또는 아카이브 정책으로 장기 보관 분리
- 권장 구현 방식:
  - 클라이언트 기록 + 서버 검증, 또는 Cloud Functions 트리거
  - 일 배치 백업을 Cloud Storage로 내보내기
  - 월 단위 자동 리포트 생성 경로 확보
- 보안/권한:
  - 관리자(또는 지정 역할)만 조회/다운로드 허용
  - 하드 삭제 대신 상태 변경(`deleted`, `rejected`) 유지
  - 이력 문서는 append-only 원칙 권장

## 6) 이관 체크리스트

- [ ] 보관 기간/접근 권한 정책 확정
- [ ] 자동 백업 스케줄 및 실패 알림 구성
- [ ] 기존 수동 백업 파일 이관 범위 결정
- [ ] 운영 매뉴얼 업데이트
